## STS3215-C001

### 基础属性

详细参数编码参照 [内存表解析](F:\A-Notes\TYpora_note\飞特舵机STS内存表解析.md)

- 默认**波特率**为1000000，串口直接和驱动板通信
- 所有舵机的**初始ID为1**，可以通过软件依次设置每个舵机的ID

![Snipaste_2025-07-19_16-58-21](F:\截屏的图片\Snipaste_2025-07-19_16-58-21.png)

- **编码器的单位**为0.087，一圈为4095个计数值，如果选择绝对圈旋转，最多支持 ± 30712(± 7.5圈)，实际使用不建议超过7圈
  - 如果为步进电机模式，则不能超过32765
- **加速度值**为 0 ~ 254 ， 单位：8.7°/s^2，系统限制最大为50，只能在0到50之间来调节
- **运行速度**的可调范围为 -32767 ~ 32767，0代表默认最大运行速度，非实时控制速度  单位：0.0146RPM (这里通过相位设置了这个最高精度的单位)
  - 如果在开环PWM调速的模式下，范围为 0-1000，没有单位，这种可以自行设计反馈系统控制
- **接线：！！！！！！非常重要**
  - 驱动板的TX和RX直接对应MCU的TX以及RX，**不需要交叉**


### 返馈数据

- 电压:当前舵机工作电压，单位：0.1V 温度：当前舵机内部工作温度，单位：°C 
- 扭矩：当前控制输出驱动电机的电压占空比，单位：0.1% 
- 电流：最大可测量电流为 500 * 6.5mA= 3250mA，单位：6.5mA  
- 位置：反馈当前所处位置的步数，每步为一个最小分辨角度；绝对位置控制方式，最大值对应最大有效角度。单位：步。 
- 目标：即目标位置，每步为一个最小分辨角度，绝对位置控制方式，最大对应最大有效角度。单位：步。 
- 移动：即移动标志，舵机在运动时标志为 1，舵机停止时为 0。
- 状态：Bit0 Bit1 Bit2 Bit3 Bit4 Bit5 对应位被置 1 表示相应错误出现
  - (Bit0 - Bit5)：电压 传感器 温度 电流 角度 过载 ，对应位 0 为无相应该错误。

### 相位权重分析

举例：

- BIT0（1）：驱动方向相位；(0)正向，(1)反向
- BIT1（2）：驱动桥模式；(0)无刷，(1)有刷，重启生效
- BIT2（4）：速度单位；(0)0.732RPM，(1)0.0146RPM
- BIT3（8）：速度模式；(0)速度0表示停止，(1)速度0表示最高速度

这里有四个BIT位，如果我设置相位为12，那么它一定是8+4，即速度模式置1(速度单位选0.0146，精度高)，速度单位置1(速度0表示最高速度)

**我目前设置为12和13，分别代表电机旋转正向和反向** 







### 运动模式

#### 1.步进模式

步进模式是基于相对位置进行的位置转动，不受角度限制。

##### 1.1.参数调整

- 首先将角度限制全部归0，利用软件调试

![Snipaste_2025-07-19_16-45-23](F:\截屏的图片\Snipaste_2025-07-19_16-45-23.png)

- 运行模式改为3

![Snipaste_2025-07-19_16-48-06](F:\截屏的图片\Snipaste_2025-07-19_16-48-06.png)

##### 1.2.使用方式

- 不断写入目标位置，这个目标位置是基于你当前的位置，然后期望继续旋转的角度

  - 一圈的编码器值是 4095，舵机反馈的角度只会储存单圈绝对值

  ![Snipaste_2025-07-19_16-51-27](F:\截屏的图片\Snipaste_2025-07-19_16-51-27.png)

  - 单次设置的最大目标值不能超过 32765，<u>编码器的单位为0.087°</u>

- 内置闭环，如果处理目前的想法是线性差值来控制角度变化的丝滑程度
- 默认速度为最大速度

####  2.开环PWM调速模式

**实际使用可以自行设计反馈系统**

##### 2.1.参数调整

- 首先设置角度限制必须为0
- 然后运行模式设置为2

##### 2.2.使用方式

- 直接设置当前速度即可，类比电流控制，利用反馈数据来进行闭环控制

![Snipaste_2025-07-19_18-17-16](F:\截屏的图片\Snipaste_2025-07-19_18-17-16.png)

- 可以利用开环PWM调速设计多圈绝对值旋转，区别于位置伺服模式，可以自主根据数据来设计整个闭环伺服系统





### 库移植后的调用

#### 1.性能优化

##### 1.1延时函数优化

- 使用定时器实现us延时，定时器配置为1MHZ实现us的步进(**F1的PSC设为71**)，同时使用向上计数模式，<u>除此之外不允许开启任何功能</u>，使用基本定时器(F1的TIM6或者TIM7)即可
- 定时器启动要确保在舵机设备初始化之前进行

```c
HAL_TIM_Base_Start(&htim6);  // 用于精确延时的定时器，确保最先初始化

// 新增微秒级延时
void DelayUs(uint32_t us) {
    __HAL_TIM_SET_COUNTER(&htim6, 0);
    while(__HAL_TIM_GET_COUNTER(&htim6) < us);

}
//FT舵机总线切换延时，时间大于10us 
void ftBus_Delay(void)
{
//	HAL_Delay(1);
  DelayUs(20);   // 改为20us，这个函数为库里面专用，只改时间
}
```
