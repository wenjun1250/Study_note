#### 1.1.初步代码解析

```python
import time
import os
import sys

from media.sensor import *
from media.display import *
from media.media import *
from time import ticks_ms  # 用来计算fps

sensor = None

try:
    print("camera_test")
    # Sensor可以指定id，庐山派开发板的默认摄像头接口为CSI2，如果Sensor默认内不指定id，则默认为id=2.
    # 这里也可以指定分辨率，防止画面变形，你输出的分辨率和传感器的分辨率要同步
    sensor = Sensor(width = 640,hight = 480) # 构建摄像头传感器实例 width = 640,hight = 480
    # 构造sensor对象后必须进行复位
    sensor.reset()

    # 鼠标悬停在函数上可以查看允许接收的参数
    # 控制输出画面大，即使摄像头初始化了高清的画布，仍然可以裁切图像来输出画面VGA
    sensor.set_framesize(Sensor.VGA)
     # 设置像素格式，颜色分别为三个通道，通常是红绿蓝，数值调整(0-255)
     # chn 为输出通道，没设备有三个输出通道
    sensor.set_pixformat(Sensor.RGB565, chn=CAM_CHN_ID_0)

    # 初始化显示设备，这里初始化的是LCD显示设备，to_ide=True是为了显示画面到ide上面
    # 正常调试不允许更换LT9611这个参数，这个是HDMI拓展屏，支持1k像素，如果换成LCD，ST7701,则无法显示这么高清的图像
    # VIRT为 IDE缓冲区显示 width = 640, height = 480,缓冲区不支持1k的像素，如需要显示1k，需要切换到LT9611参数
    Display.init(Display.VIRT, to_ide=True)
    # 初始化媒体管理器，即上述设备全部初始化
    MediaManager.init()
    # 启动 sensor
    sensor.run()
    clock = time.clock()

    while True:
        clock.tick()
        # 是一个“检查点”，它会定期检查用户是否在开发环境中点击了“停止”按钮,方便打断
        os.exitpoint()
        img = sensor.snapshot(chn=CAM_CHN_ID_0)
        # 画线 thickness 为 粗些调节
     	# 正常画面的坐标是以左上角为原点
        img.draw_line(120, 50, 300, 130, color=(0, 0, 255), thickness=2)

        # 绘制方框，参数依次为：x, y, w, h, 颜色，线宽，是否填充 fill 为中间是否填充
        img.draw_rectangle(80, 200, 100, 100, color=(0, 0, 255), thickness=4, fill=False)

        # 绘制关键点, 列表[(x, y, 旋转角度)] 260,240为点的坐标角度，100为距离x轴的角度，
        # 可以同时绘制多个关键点，用列表储存多个列表进行嵌套，每个子列表储存坐标信息以及画线的角度
        img.draw_keypoints([[260, 240, 100],[140,140,90]], color=(255, 255, 0), size=30, thickness=2, fill=False,size = 30)

        img.draw_string_advanced(50, 50, 50, "fps: {}".format(clock.fps()), color=(255, 0, 0))
        Display.show_image(img)


except KeyboardInterrupt as e:
    print("用户停止: ", e)
except BaseException as e:
    print(f"异常: {e}")
finally:
    if isinstance(sensor, Sensor):
        sensor.stop()
    Display.deinit()
    os.exitpoint(os.EXITPOINT_ENABLE_SLEEP)
    time.sleep_ms(100)
    MediaManager.deinit()
```

##### 1.1.1.FPS计算

- 首先移植时钟对应模块： **`from time import ticks_ms`**  # 用来计算fps
- 创建时钟实例： **`clock = time.clock()`** 
- 根据这个实例，使用计数器函数： **`clock.tick()`** 自动计算时间周期间隔
- 直接通过函数来获取FPS值： **`clock.fps()`** \
- 打印在屏幕上方便调试： **`img.draw_string_advanced(50, 50, 50, "fps: {}".format(clock.fps()), color=(255, 0, 0))`**
  - 前面三个参数的顺序依次是 **X坐标 Y坐标  线条粗细**


##### 1.1.2.摄像头以及画面显示

- **1.**首先构建摄像头实例，方便后续使用摄像头的函数： **`sensor = Sensor(width = 640,hight = 480)`** 
  - **`width = 640,hight = 480`** ：这部分是初始化摄像头的分辨率，如果你后续处理的话，分辨率越小处理速度越快
  -  **`Sensor`**  可以指定id，庐山派开发板的默认摄像头接口为CSI2，如果Sensor默认内不指定id，则默认为id=2.
    - 如果需要同时显示多个屏幕，那么其他屏幕需要指定id

- **2.**可以控制输出画面的大小：**`sensor.set_framesize(Sensor.VGA)`** ：
  - **`Sensor.VGA`** ：为分辨率 640 * 480, 同时你也可以自定义，使用 **`width = 640,hight = 480`** 这种方式来自定义分辨率
  - 这里的分辨率需要和摄像头的初始分辨率相同，否则会造成画面被拉伸，造成失真
- **3.**同时可以继续设置画面的格式： **`sensor.set_pixformat(Sensor.RGB565, chn=CAM_CHN_ID_0)`** 
  - **`chn=CAM_CHN_ID_0`** : 为通道号，只需要显示一个屏幕的时候，指定其中一个输出通道就行
  - 常见的格式包括： **`Sensor.RGB565`** **`Sensor.GRAYSCALE`**  
    - **`Sensor.RGB565`** : 为彩色图像，如果对图像处理速度会相对较慢，二值化的方式通常采用 **`LAB`** 阈值
    - **`Sensor.GRAYSCALE`**  ： 为灰度图像，只有亮度数据，处理速度快于彩色图像，通过灰度阈值进行二值化
- **4.**需要设置显示方式初始化： **`Display.init(Display.VIRT, to_ide=True)`** 
  - VIRT 为虚拟环境，代表不输出外接设备，输出到 IDE 的缓存区，同时确保 **`to_ide=True`** 
  - 拓展屏为 **`ST7701`** , HDMI输出拓展屏为 **`LT9611`** ，在 **`VIRT`** 的初始模式下，IDE 不支持1k分辨率，推荐 **`Sensor.VGA`** 分辨率
- **5.**然后需要进行一次全面媒体设备初始化，保证所有媒体模块能够起作用： **`MediaManager.init()`** 
  - **`from media.media import * `** ：这种模块的导入方法，代表你可以使用该模块的所有内置函数
- **6.**最后需要启动摄像头传感器： **`sensor.run()`** 
  - 针对异常情况可以使用 **`sensor.stop()`** 来关闭摄像头
- **7.** 对于画面需要指定通道号去获取: **`img = sensor.snapshot(chn=CAM_CHN_ID_0)`** 
  - **`sensor.snapshot`** : 代表截取一帧
  - **`chn=CAM_CHN_ID_`** : 这里依据前面设置输出画面的指定通道，默认为0
- **8.**然后将这一帧画面显示出来： **`Display.show_image(img)`** 
  - 针对异常情况，可以使用 **`Display.deinit()`** 直接关闭显示

##### 1.1.3.异常情况处理

- 异常处理板块

```python
except KeyboardInterrupt as e:
    print("用户停止: ", e)
except BaseException as e:
    print(f"异常: {e}")
finally:
    if isinstance(sensor, Sensor):
        sensor.stop()
    Display.deinit()
    os.exitpoint(os.EXITPOINT_ENABLE_SLEEP)
    time.sleep_ms(100)
    MediaManager.deinit()
```

- 首先要确保 while 循环里面有  **`os.exitpoint()`** ,作为一个“检查点”，它会定期检查用户是否点击了“停止”按钮,方便打断
- **`except KeyboardInterrupt as e:`** 这是一个异常处理语法，用户如果在IDE里面点击了停止按钮，**`os.exitpoint()`** 会将他捕获到，触发 **`KeyboardInterrupt`** 异常，将这个异常赋值给 **`e（可以改其他变量名）`** ， 并且将异常打印出来
  - 针对这些错误情况，可以通过打印来查看问题，下面会介绍一些常用错误情况
    - **`ZeroDivisionError`**    除以零
    - **`FileNotFoundError`**    文件不存在
    - **`TypeError`**      类型不匹配
    - **`ValueError`**      值不合法
- 触发异常后则继续执行到 **`finally`** 板块，首先判断 **`sensor`** 是否指向 **`Sensor`**， 防止空指针，然后将摄像头关闭
- 继续关闭显示模块，**`Display.deinit()`** 
- 控制主循环进行休眠，此时进入低功耗模式，**`os.exitpoint(os.EXITPOINT_ENABLE_SLEEP)`** 
- 然后最后给出100ms的时间，等待上述操作完成
- 针对媒体资源，我们手动进行释放： **`MediaManager.deinit()`** 
- 针对 CANMV 这种嵌入式开发，**必须手动关闭摄像头、显示、DMA、缓存等硬件资源，防止系统崩溃**

##### 1.1.4.图像绘制

- **首先，依据一个原则，即左上角的顶点为原点， 如果不是，就调整一下画面，或者根据情况里更换原点处理**
- 绘制线段： **`img.draw_line(120, 50, 300, 130, color=(0, 0, 255), thickness=2)`** 
  - **120, 50** : 为起点的 **`x`** 坐标和 **`y`** 坐标，
  - **300, 130**: 为终点的 **`x`** 坐标和 **`y`** 坐标，
  - **`colo`** 为你的颜色格式，这里设置的颜色为**RGB888**格式，内部兼容RGB565图像格式的颜色显示，自动转换
  - **`thickness`** ： 线段的粗细调节
- 绘制方框： **`img.draw_rectangle(80, 200, 100, 100, color=(0, 0, 255), thickness=4, fill=False)`** 
  - **80, 200**: 为绘制点的 **`x`** 坐标和 **`y`** 坐标 
  - **100, 100**: **代表方框的宽和高**，这里是依据固定原点来判断的，<u>比如左上角为原点，那么垂直向下为 **y** 的正方向，水平向右为 **x** 的正方向，宽朝着 **x** 的正方向延申，长朝着 **y** 的正方向延申</u>
  - **`thickness`** : 绘制的线条粗细
  - **`fill = False`**: 是否填充方框，一般不填充

  - 可以同时绘制多个关键点，用列表储存多个列表进行嵌套，每个子列表储存坐标信息以及画线的角度
    - 以 **`[[260, 240, 100],[140,140,90]]`** 为例，这里分别以固定坐标画原点，每个列表的参数顺序为 **`x y angle`** 
    - **`angle`** 为 固定的偏离 x正轴的角度

```python
    while True:
        # 获取当前这一帧的图像
        img = sensor.snapshot(chn=CAM_CHN_ID_0)
		# 裁切图像，缩小范围，方便二值化
        img = img.copy(roi=(366,188,120,160))
        # 图像转化为灰度图像 注意这里必须加入参数 copy=True,否则会修改原图像
        # 灰色图像二值化处理速度快，方便特征检测
        img_rect = img.to_grayscale(copy=True)
        # 灰色图像二值化，阈值为固定阈值，可以通过IDE里面直接调试出来
        img_rect = img_rect.binary([(80, 158)])
        # 矩形检测函数，传入的参数为阈值，低于这个阈值的像素点矩形不会纳入检测 
        # roi=Auto，默认有效区域为全局，同时你也可以不裁剪，然后直接设置有效区域即可，但是裁剪后方便二值化调试参数
        # 这个函数会返回一个列表，这个列表里面包含了所有你检测到的矩阵对象，如果是三个矩阵，那么这个列表就包含三个对象
        rects = img_rect.find_rects(threshold=10000)
        # 遍历所有的对象，分别绘制出边框
        for rect in rects:          
            # 找出各个定点的坐标，然后利用点和点之间画直线，可以绘制方框
            # 导出这个对象的嵌套列表，一个列表包含四个子列表，每个子列表包含X Y 坐标，然后根据坐标绘制出边框
            # [[x1,y1],[x2,y2],[x3,y3],[x4,y4]]
            corner = rect.corners()
            img.draw_line(corner[0][0], corner[0][1], corner[1][0], corner[1][1], color=(0, 255, 0), thickness=2)
            img.draw_line(corner[2][0], corner[2][1], corner[1][0], corner[1][1], color=(0, 255, 0), thickness=2)
            img.draw_line(corner[2][0], corner[2][1], corner[3][0], corner[3][1], color=(0, 255, 0), thickness=2)
            img.draw_line(corner[0][0], corner[0][1], corner[3][0], corner[3][1], color=(0, 255, 0), thickness=2)
		# 压缩图像到IDE上面，这里是在原彩色图像上进行绘制，否则无法绘制彩色轮廓

        Display.show_image(img_rect)

  - 这个函数会返回一个列表，这个列表里面包含了所有你检测到的矩阵对象，如果是三个矩形，那么这个列表就包含三个对象

##### 1.2.2.二值化配合线段检测优化

**代码解析:**

```python
while True:
    clock.tick()
    # 是一个“检查点”，它会定期检查用户是否在开发环境中点击了“停止”按钮,方便打断
    os.exitpoint()
    img = sensor.snapshot(chn=CAM_CHN_ID_0)

    # 复制一份图像
    img_line = img.to_rgb565(copy=True)
#        img = img.copy(roi=(210,164,296,296))
    img_line.midpoint_pool(2, 2)
    lines = img_line.find_line_segments((0, 0, 1024//2, 768//2), 15, 15)
    for line in lines:
        if line.length() > 40:
            img.draw_line(line.x1()*2, line.y1()*2, line.x2()*2, line.y2()*2, color=(0, 255, 0), thickness=5)
            
    img.compressed_for_ide()
    Display.show_image(img)
```

- 首先复制一份彩色图像进行处理，原图像用来显示: **`img_line = img.to_rgb565(copy=True)`** 

- 然后对图像进行压缩，即中值滤波，可以减少图像处理时间：**`img_line.midpoint_pool(2, 2)`** 

  - 参数为: **`xScale, yScale`** ，分别代表 X 方向和 Y方向的中值滤波个数，如果 **`xScale`** 设为5，则代表每五个像素点取一个中点
  - 可以减少图像噪点，适当模糊画面，也可以加强二值化效果
  - **使用细节：** 后续你所绘制的线段是显示在原图像上的，所以如果 **`xScale`** 设为2，后续的坐标和宽高都需要乘以2，线段函数则需要对扫描区域除以设置的 **`xScale`** 或 **` yScale`** 的值
    - 可以用来解决该函数的分辨率适用性，该函数对于超过320分辨率的效果会降低

- 查找线段： **`lines = img_line.find_line_segments((0, 0, 1024//2, 768//2), 15, 15)`** 

  - **（15，15）**： 
    - **`merge_distance=15`**       两线段中心点相距小于15像素则合并        
    - **`max_theta_diff=15`**         # 两线段角度差小于15°则合并
  - 参数顺序：起点坐标 **X Y**， 图像大小 **宽 高** 
  - 由于我的代码执行了 **`img_line.midpoint_pool(2, 2)`**  所以需要对扫描区域进行倍数减小，**`1024//2`** 为整除2，防止小数出现

- 绘制线段：

  - ```python
    for line in lines:
        if line.length() > 40:
            img.draw_line(line.x1()*2, line.y1()*2, line.x2()*2, line.y2()*2, color=(0, 255, 0), thickness=5)
    ```

  - 循环列表里的所有线段对象：**`for line in lines:`** 

  - 手动增加阈值，减小误判： **`if line.length() > 40:`** 

  - **`img.draw_line`** 绘制线段

##### 1.2.3图像优化处理函数

- **`img_line.midpoint_pool(2, 2)`** ： 用来中值滤波，保留边缘轮廓压缩分辨率，增强抗噪
- **`max_pool(2, 2)`** ： 用来突出最亮的值，强调高亮边缘，可以用于处理高亮轮廓
  - 适用场景包括 **工科省赛标靶识别**  **激光追踪**  
- **`histeq()`** ： 直方图均衡化，<u>容易过曝</u> ,目前仅考虑到工科省赛标靶识别
- **`gamma_corr`** ： 伽马校正
  - `gamma < 1`：提升暗部亮度，整体变亮
  - `gamma = 1`：无变化（线性）
  - `gamma > 1`：压暗图像，对比度增强

```python
data = uart.read(128)  # 读取最多 128 字节

def uart_analys(data)
	if len(datas) >= x:    # 这里的意义在于防止发生通讯错误，即x的长度必须要达到完整要求
    # 格式字符串 '<I f H B' 对应：
    # - I: uint32_t (4字节)
    # - f: float (4字节)
    # - H: uint16_t (2字节)
    # - B: uint8_t (1字节)
    	sensor_id, temp, voltage, status = struct.unpack('<IfHB', data) # 解包
    # 如果发送端发送16进制，那么接收端就会变成10进制
    # 发送的数据不足一帧的时候会报错，根据你的选择来解析，所以需要在定时器里面做解析，会稍显复杂
while True:

        datas = uart.read(128)  # 一次性最多读 128 字节

        if len(datas) >= 1:

- **`ustruct.unpack`** : 如果仅仅只有一个变量的话，他会返回一个元组，而只有大于一个变量接收的时候，变量接收才正常，比如解包一个**`int`** 类型就可以正常识别
  - <u>解决这个问题就需要多增加一个 **`,`** 符号，即可解包元组</u>

- 总之利用数据结构的方式在数据接收的时候根据数据类型来设置解包格式

      bottom = int(round(dh + 0.1))  # 底部填充像素数

    - **`dw`** 和 **`dh`** 分别为计算出来的需要填充的水平填充量和垂直填充量

                      # 将展平的输出添加到 results 列表。
                      results.append(result)
                  # 触发垃圾回收，释放未使用的内存。
                  gc.collect()
                  # postprocess
                  if model_type == "AnchorBaseDet":
                      # 解码模型输出，生成检测框的坐标，det_boxes为列表，包含了检测框的坐标(基于 640 x 640)
                      # 解码自动将模型图片尺寸缩放到输入图像的尺寸，定义好的尺寸
                      det_boxes = aicube.anchorbasedet_post_process( results[0], results[1], results[2], kmodel_frame_size, frame_size, strides, num_classes, confidence_threshold, nms_threshold, anchors, nms_option)
                  elif model_type == "GFLDet":
                      det_boxes = aicube.gfldet_post_process( results[0], results[1], results[2], kmodel_frame_size, frame_size, strides, num_classes, confidence_threshold, nms_threshold, nms_option)
                  else:
                      det_boxes = aicube.anchorfreedet_post_process( results[0], results[1], results[2], kmodel_frame_size, frame_size, strides, num_classes, confidence_threshold, nms_threshold, nms_option)
                  # 清空 OSD 图像（800×480，ARGB8888），准备绘制新帧。
                  osd_img.clear()
                  if det_boxes:
                      for det_boxe in det_boxes:
                          x1, y1, x2, y2 = det_boxe[2],det_boxe[3],det_boxe[4],det_boxe[5]
                          # 开始映射，即模型输出的 640 x 640 映射到 800 * 480,w和h分别为映射后的宽度和高度
                          w = float(x2 - x1) * DISPLAY_WIDTH // OUT_RGB888P_WIDTH
                          h = float(y2 - y1) * DISPLAY_HEIGHT // OUT_RGB888P_HEIGH
                          # x1 * DISPLAY_WIDTH // OUT_RGB888P_WIDTH 这种开始对坐标也进行映射
                          osd_img.draw_rectangle(int(x1 * DISPLAY_WIDTH // OUT_RGB888P_WIDTH), int(y1 * DISPLAY_HEIGHT // OUT_RGB888P_HEIGH) , int(w) , int(h) , color=color_four[det_boxe[0]][1:])
                          label = labels[det_boxe[0]]
                          score = str(round(det_boxe[1],2))
                          osd_img.draw_string_advanced( int(x1 * DISPLAY_WIDTH // OUT_RGB888P_WIDTH) , int(y1 * DISPLAY_HEIGHT // OUT_RGB888P_HEIGH)-50,32, label + " " + score , color=color_four[det_boxe[0]][1:])
                  Display.show_image(osd_img, 0, 0, Display.LAYER_OSD3)
                  gc.collect()
              rgb888p_img = None
  ```
