## K210策略规划

> 电赛E题提供的思路见解，对于需要处理多个元素:  单次图像捕获，多任务处理， 即对于矩形提取可以直接对他进行二值化，然后进行边缘检测得到矩形的坐标，同时可以通过物理校正来帮助图像处理，而其他激光颜色则也可以通过颜色阈值来进行二值化找到激光的位置，实现闭环控制

### 1.帧率获取和显示

#### 1.1.局部代码展示

```python
import sensor, image, time, lcd   # 导入模块
import utime					  # 导入软件定时器
from machine import Timer         # 导入硬件定时器

lcd.init()                          # 初始化模块
lcd.clear(lcd.RED)                  # 用红色来清除屏幕

# sensor.reset(dual_buff=True) # improve fps
#sensor.reset()                      # Reset and initialize the sensor.
sensor.reset(freq=24000000, dual_buff=True) # 开启双缓冲模式，可以提高帧率，这里帧率在45比较极限，
                                            # 默认频率为24MHZ,如果画面质量不好，可以适当调低频率
sensor.set_pixformat(sensor.RGB565) # Set pixel format to RGB565 (or GRAYSCALE)
                                    # 画面图像为RGB565格式
sensor.set_framesize(sensor.QVGA)   # Set frame size to QVGA (320x240)
                                    # 设置画面分辨率，QQVGA为160x120
sensor.skip_frames(time = 2000)     # Wait for settings take effect.
                                    # 跳过初始2000ms内的图像，保证图像传输的稳定性
clock = time.clock()                # Create a clock object to track the FPS.

# 全局变量，这里为计算fps的变量
frame_count = 0
fps = 0.0

# 定时器回调，每 500ms 计算帧率，相比于clock.fps(),更加精准
def calculate_fps(timer):  # 这里必须传入参数timer
    global frame_count, fps
    fps = frame_count * (1000 / 500)  # 帧率 = 帧数 × (1000ms / 周期)
    frame_count = 0

# 设置定时器，周期 500ms
timer = Timer(Timer.TIMER0, Timer.CHANNEL0, mode=Timer.MODE_PERIODIC, period=500, callback=calculate_fps)

while(True):
    clock.tick()                    # Update the FPS clock.
    img = sensor.snapshot()         # 捕获一帧图像
    frame_count += 1
    img.draw_string(0, 0, "FPS: {:.2f}".format(fps), color=(255, 0, 0))  # 显示帧率

    lcd.display(img)                # 将捕获的图像显示在lcd屏幕上
```

#### 1.2.函数解析

##### 1.2.1.摄像头传感器模块

- **`sensor.reset`**: 如果不传入任何参数，默认26MHz，无双缓冲，一般为了提升帧率会开启双缓冲模式

  - 下面为该函数的接口，一般只需要修改时钟频率和是否开启双缓冲即可，时钟频率推荐20到24MHZ，极限可达36MHZ

    ```python
    def reset(
        freq=26000000,           # 时钟频率（Hz），默认 26MHz
        set_regs=True,          # 初始化传感器寄存器
        dual_buff=False,        # 启用双缓冲
        sensor_type=None,       # 传感器类型（None 为自动检测，如 OV2640）
        reset_pin=None,         # 复位引脚（通常自动配置）
        power_pin=None,         # 电源引脚
        i2c=None,               # I2C 接口（默认自动）
        xclk_freq=None          # 外部时钟频率（通常由 freq 决定）
    ):
    ```

- **`sensor.set_pixformat(sensor.RGB565)`** ： 设置图像的格式，如果是彩色推荐RGB565格式，如果是灰度图像则为 **`sensor.GRAYSCALE`** 格式

- **`sensor.set_framsize(sensor.QVGA)`** ： 设置画面分辨率

  - **`sensor.QVGA`**（预定义常量）: 320x240

  - **`sensor.QQVGA`**：160x120（19,200 像素）。

  - **`sensor.VGA`**：640x480（307,200 像素）。<u>此为最大支持分辨率</u>

  - 不支持分辨率自定义，但是可以裁切图像，

    - ```python
      img = img.crop((60, 45, 200, 150))  # 从 (60,45) 裁剪 200x150
      ```

- **`sensor.skip_frames(time = 2000)`** ：设法跳过2000ms内的帧数，让相机图像在改变相机设置后稳定下来
  - **`(n,time)`** 一共两个参数：
    - 若 **未指定 n 和 time**：默认跳过 **300 毫秒**的帧（基于当前帧率）。
    - 若 **仅指定 n**：跳过 n 帧，时间取决于帧率（例如，10 FPS 下，10 帧 = 1 秒）。
    - 若 **仅指定 time**：跳过 time 毫秒内的帧，帧数取决于帧率（例如，10 FPS 下，2000ms = 20 帧）。
    - 若 **n 和 time 皆指定**：跳过 n 帧，但保证至少等待 time 毫秒后返回（若 n 帧耗时不足 time，则额外延时）。
- **`sensor.snapshot()`**  : 捕获一帧图像

##### 1.2.2.硬件定时器

- K210 包含 3 个硬件定时器模块：TIMER0、TIMER1、TIMER2。
  - 每个模块有 4 个独立通道：CHANNEL0 到 CHANNEL3。总计：3 × 4 = 12 个独立定时器通道。
  - 通道模式支持:
    - **周期模式**（Periodic）：定期触发（如你的 500ms 周期）。支持定时中断调用回调函数
    - **单次模式**（One-Shot）：触发一次后停止。
    - **PWM 输出**：生成脉宽调制信号。

- **`Timer(Timer.TIMER0,Timer.CHANNEL0, mode=Timer.MODE_PERIODIC, period=500,callback=calculate_fps)`**:
  - **`Timer.TIMER0`**：选择第一个定时器模块（K210 有 3 个）。
  - **`Timer.CHANNEL0`**：选择模块内的第一个通道（每个模块 4 个通道）。
  - **`mode=Timer.MODE_PERIODIC`**：周期模式，持续每 500ms 触发。
  - **`period=500`**：周期 500 毫秒（0.5 秒）。
  - **`callback=calculate_fps`**：触发时执行的函数，指定一个函数为定时周期中断回调函数。这个函数必须传入参数(timer)
- **`timer.deinit() `** ；停止定时器

##### 1.2.3.lcd屏幕

- **`img.draw_string`** ： 绘制字符串
  - 详细参数：
  - **`x`**（整数）：文本左上角的水平坐标（你的代码：0，图像最左侧）。
  - **`y`**（整数）：文本左上角的垂直坐标（你的代码：0，图像最顶部）。
  - **`text`**（字符串）：要绘制的文本。**`"FPS: {:.2f}".format(fps)，`**格式化字符串，显示帧率（例如 "FPS: 12.34"）。
    {:.2f}：保留 2 位小数。
    - **`{:.2f}`** ：占位符，表示插入一个保留 2 位小数的浮点数
    - **` .format(fps)`** ： 将括号内的变量格式化为前面的占位符，方便插入
  - **`color`**（元组或整数，可选）：文本颜色。
    - RGB565 模式：(R, G, B)，范围 0-255（你的代码：(255, 0, 0)，红色）。
    - GRAYSCALE 模式：整数（0-255，亮度），例如 255（白色）。
  - **`scale`**（浮点数，可选）：**字体缩放比例**，默认 1.0。
    - 例如，scale=2 放大字体两倍。
  - **`mono_space`**（布尔值，可选）：是否使用等宽字体，默认 False（非等宽）。
  - **`char_rotation`**（整数，可选）：字符旋转角度（度），默认 0。
  - **`char_hmirror`**（布尔值，可选）：水平镜像，默认 False。
  - **`char_vmirror`**（布尔值，可选）：垂直镜像，默认 False。

- **`lcd.display(img) `**    : 将捕获的图像显示在lcd屏幕上
- **`lcd.clear(lcd.RED)`** :  用红色来清除屏幕

##### 2.1.1.IO口映射，中断配置

```python
from fpioa_manager import fm
from maix import GPIO

fm.register(board_info.BOOT_KEY,fm.fpioa.GPIOHS2,force = True) # force为强制注册，忽略冲突
key = GPIO(GPIO.GPIOHS1, GPIO.IN, GPIO.PULL_NONE)               # 配置该IO口下拉输入，即IO口默认低电平
 # 配置中断，下降沿检测，即按键回弹的时候触发,Image_photo为中断触发的回调函数
key.irq(Image_photo, GPIO.IRQ_FALLING, GPIO.WAKEUP_NOT_SUPPORT, 7)
```

- **`board_info.BOOT_KEY`** ：野火导入的 **`BOOT`** 引脚的IO口,默认绑定上拉输入
- **`key = GPIO(GPIO.GPIOHS1, GPIO.IN, GPIO.PULL_UP)`**: 配置IO口为输入模式以及下拉模式
  - **`GPIO_IN`**: 输入模式，可以切换**`GPIO_OUT`**  模式
  - **`GPIO.PULL_UP`**: 上拉模式，默认高电平，可以切换下拉模式，即 **`GPIO.PULL_NONE`** 模式
- **`key.irq(Image_photo, GPIO.IRQ_FALLING, GPIO.WAKEUP_NOT_SUPPORT, 7)`**: 配置IO口中断
  - **`Image_photo`** : 为用户指定的回调函数
  - **`GPIO.IRQ_FALLING`**: IO口触发中断的方式，这里是下降沿触发，可以改成 **` GPIO.IRQ_RISING`** 上升沿触发
  - **`7`** : 中断优先级，0-7，7为最高优先级

#### 2.5.互斥锁

```python
import _thread      # 导入多线程模块

lock = _thread.allocate_lock()  # 创建互斥锁
buffer = []  # 共享缓冲区

# 模拟一个任务
def image_processing_thread(): 
    while True:
        data = b"image_data"  # 模拟图像处理
        with lock:  # 获取锁，锁自动进行释放管理，所以对于防止冲突缓冲区写入可以实现这一操作
            buffer.append(data)  # 写入缓冲区
        time.sleep(0.1)  


```

- **`buffer.append(data)`**:   写入缓冲区,使用列表的内置函数


#### 2.6.进阶语法

- **for的递推语法**

  ```python
  ready_tasks = [t for t in tasks if t.counter >= t.period]
  ```

  - **`ready_tasks`**: 一个列表

  - **`t`** 为对象，这里从 **`tasks`** 中 遍历，这里是首先遍历一遍创建一个列表查看是否满足 **` t.counter >= t.period`** 这个条件，然后继续遍历一遍，将符号条件的任务重新新建一个列表

  - 等价语法:

    ```python
    ready_tasks = []
    for t in tasks:
        if t.counter >= t.period:
            ready_tasks.append(t)
    ```

- **min函数**

  ```python
  selected = min(ready_tasks, key=lambda t: t.priority if not t.urgent else -1)
  ```

  - 传入一个列表 **`ready_tasks`** ，按照 **`key`** 的规则来找到最小的值

  - **`key=lambda t: t.priority if not t.urgent else -1`** : **lambda表达式**

    - **`t`**  是 **`ready_tasks`** 中的一个任务对象。

    - 如果 **`t.urgent == False`**，就使用 **`t.priority`** 来作为排序依据。

    - 如果 **`t.urgent == True`**，则返回 **-1**，表示“最高优先级”。

    - 等价语法：

      ```python
      def get_key(t):
          if t.urgent:
              return -1
          else:
              return t.priority
      ```

- **字典：**

  ```python
  # 按优先级分组
  #priority_groups = {}：初始化一个空字典 priority_groups，用于存储任务分组。
  # 键是优先级（整数），值是对应优先级下任务的列表。
  priority_groups = {}  
  for task in ready_tasks:
      if task not in urgent_queue:
          # 如果当前任务的 priority 不在 priority_groups 字典的键中，则为该优先级创建一个空列表。
          # 即对当前当前任务的优先级去创建一个键，针对这个键添加当前的任务
          if task.priority not in priority_groups:
              priority_groups[task.priority] = []
          priority_groups[task.priority].append(task)
  ```

- **`priority_groups = {}`** : 新建一个字典，用字典来管理优先级顺序

- **`priority_groups[task.priority] = []`** : 为每个优先级创立一个键，方便后面的优先级索引

#### 2.7.仿RTOS的状态机设计

> 拍照保存的时间为8ms左右 抓取一帧图片 2ms  绘制字符串 0.8ms

- 对每个任务进行定时器任务的时候需要去查看每个任务的任务延时

- ```python
  # 任务类
  class Task:
      def __init__(self, func, priority, period):
          self.func = func          # 任务函数
          self.priority = priority  # 优先级（0 最高）
          self.period = period      # 周期（ms，10ms 的倍数）
          self.counter = 0          # 时间计数器
          self.urgent = False       # 紧急标志
          self.timestamp = 0        # 加入紧急队列的时间戳
  
  # 任务列表和紧急队列
  tasks = []
  urgent_queue = []
  
  # 状态机核心函数
  def state_machine(timer):
      global tasks, urgent_queue
      # 更新所有任务计数器
      for task in tasks:
          task.counter += 5   # 单词计数，可以根据你需要设计的定时器最小中断周期来算
  
      # 处理紧急队列（优先级最高）
      if urgent_queue:
          selected = min(urgent_queue, key=lambda t: (-1 if t.urgent else t.priority, t.timestamp))
          selected.func()
          selected.counter = 0
          if selected.urgent:
              selected.urgent = False
              urgent_queue.remove(selected)
  
      # 筛选符合周期条件的任务
      ready_tasks = [t for t in tasks if t.period > 0 and t.counter >= t.period]
      if ready_tasks:
          # 按优先级分组
          #priority_groups = {}：初始化一个空字典 priority_groups，用于存储任务分组。
          # 键是优先级（整数），值是对应优先级下任务的列表。
          priority_groups = {}  
          for task in ready_tasks:
              if task not in urgent_queue:
                  # 如果当前任务的 priority 不在 priority_groups 字典的键中，则让优先级作为字典的键，值为任务列表
                  # 即对当前当前任务的优先级去创建一个键，针对这个键添加当前的任务
                  if task.priority not in priority_groups:
                      priority_groups[task.priority] = []
                  priority_groups[task.priority].append(task)
  
          # 按优先级从高到低执行 sorted() 函数将字典的键按升序排序
          for priority in sorted(priority_groups.keys()):
              # 获取当前优先级下的任务列表
              tasks_at_priority = priority_groups[priority]
              if tasks_at_priority:
                  # 按时间戳排序并执行所有任务
                  for task in sorted(tasks_at_priority, key=lambda t: t.timestamp):
                      task_start = time.ticks_ms()
                      task.func()
                      task.counter = 0
                      task.timestamp = time.ticks_ms()
  
  
  # 设置紧急任务,task_id 为 任务列表索引，按照任务的注册顺序索引
  def set_urgent(task_id):
      global tasks
      for task in tasks:
          if task == tasks[task_id]:
              task.urgent = True
              task.timestamp = time.ticks_ms()
              if task not in urgent_queue:
                  urgent_queue.append(task)
              break
  
  #任务注册
  def task1():
      print("Task 1 (Priority 1, Period 20ms) executed at", time.ticks_ms())
  
  def task2():
      print("Task 2 (Priority 0, Period 10ms) executed at", time.ticks_ms())
  
  def task3():
      print("Task 3 (Priority 2, Period 30ms) executed at", time.ticks_ms())
  
  def urgent_task1():
      print("Urgent Task 1 (Priority 3) executed at", time.ticks_ms())
  
  def urgent_task2():
      print("Urgent Task 2 (Priority 4) executed at", time.ticks_ms())
  
  # 任务注册
  tasks.append(Task(task1, 1, 20))  # 优先级 1，周期 20ms
  tasks.append(Task(task2, 0, 10))  # 优先级 0，周期 10ms
  tasks.append(Task(task3, 2, 30))  # 优先级 2，周期 30ms
  urgent1 = Task(urgent_task1, 3, 0)  # 优先级 3，普通周期不执行
  urgent2 = Task(urgent_task2, 4, 0)  # 优先级 4，普通周期不执行
  tasks.append(urgent1)
  tasks.append(urgent2)
  
  # 定时器配置（5ms 周期）
  timer = Timer(Timer.TIMER0, Timer.CHANNEL0, mode=Timer.MODE_PERIODIC, period=5, callback=state_machine)
  ```

- **状态机设计:**

  - ```python
    # 全局变量
    state = 0  # 状态机状态：0=待机，1=检测到按下，2=稳定执行
    last_state = 0  # 上次状态
    last_trigger = 0  # 上次触发时间
    debounce_time = 50  # 消抖时间（ms）
    stable_time = 150  # 稳定执行时间（ms）
    
    # 状态机更新函数,用于定时器周期中断
    def IOtimer_callback(timer):
        global state, last_state, last_trigger
        # 获取当前时间戳
        current_time = time.ticks_ms()
        #检测按键状态 0=松开，1=按下
        current_value = key.value()
        if state == 0:  # 待机状态
            if current_value == 0 and last_state == 1:  # 检测到下降沿（按下）
                last_trigger = current_time
                state = 1  # 进入消抖状态
    
        elif state == 1:  # 消抖状态
            if time.ticks_diff(current_time, last_trigger) >= debounce_time:  # 50ms 消抖时间
                if current_value == 0:  # 下降沿确认
                    last_trigger = current_time
                    state = 2  # 进入稳定执行
                else:
                    state = 0  # 抖动未稳定，回到待机
    
        elif state == 2:  # 稳定执行状态
            if time.ticks_diff(current_time, last_trigger) >= stable_time:  # 稳定时间，用来做超时计算
                print("Button press confirmed at", time.ticks_ms())  # 下降沿触发
                state = 0  # 超时回到待机
    
        last_state = current_value  # 更新上次状态
    
    # 设置定时器，周期 20ms，状态机检测
    io_timer = Timer(Timer.TIMER2, Timer.CHANNEL0, mode=Timer.MODE_PERIODIC, period=20, callback=IOtimer_callback)
    ```

  - **`io_timer`**: 设置定时器，同时这个变量名不能和其他定时器搞混

  - 整体任务是基于state的标志位来进行任务设计的

#### 3.2.色块函数详解

- ```python
  image.find_blobs(thresholds, roi=Auto, x_stride=2, y_stride=1, 
                   invert=False, area_threshold=10, pixels_threshold=10, 
                   merge=False, margin=0, threshold_cb=None, merge_cb=None)  # 通过LAB阈值去找色块
  ```

  - 此函数内部自动转换到 LAB 空间
  - **`thresholds`**: 颜色阈值列表，通常为 LAB 或 RGB 空间的阈值，例如 [(L_min, L_max, A_min, A_max, B_min, B_max)]。
  - **`roi`**: 感兴趣区域（Region of Interest），默认为整个图像 (Auto)。
  - **`x_stride, y_stride`**: 搜索时的像素步幅，整数，表示跳过多少像素进行一次判断，权衡计算速度和精度
  - **`invert:`** 是否反转阈值（False 表示匹配阈值内的像素，True 表示匹配阈值外的像素）。
  - **`area_threshold`**: 最小区域面积阈值，过滤小区域。
  - **`pixels_threshold`**: 最小像素数阈值，过滤小区域。
  - **`merge`**: 是否合并相邻区域（True 时合并靠近的 blobs）。
  - **`margin`**: 合并时的边界扩展（像素）。两个色块之间的最大距离，超过此值则不会合并
  - **`threshold_cb, merge_cb`**: 回调函数，用于自定义阈值或合并逻辑。<u>这里的使用有待研究</u>

- **返回值：** 函数的返回值是一个 列表，列表中的每个元素是一个 **`blob`** 对象。每个 **`blob`** 对象描述一个检测到的连通区域（即一团符合阈值的像素）。如果没有检测到任何区域，返回空列表 **`[]`**
  - **`cx()`**: 区域中心的 x 坐标。
  - **`cy()`**: 区域中心的 y 坐标。
  - **`w()`**: 区域的宽度。
  - **`h()`**: 区域的高度。
  - **`pixels()`**: 区域的像素数。
  - **`area()`**: 区域的面积（像素数乘以密度）。
  - **`rotation()`**: 区域的旋转角度（如果适用）。
  - **`code()`**: 区域的颜色编码（如果使用多个阈值）。
  - **`count()`**: 合并后的子区域数（如果 merge=True）。

##### 3.2.1.LAB阈值

- 调试参数，一般可以通过K210拍照后来电脑上调参会比较方便，调试参考网址: [LAB在线调参网址](https://wiki.sipeed.com/threshold)

- 阈值调试经验:

  - **`L`**: 亮度参数，表示像素的明暗程度，范围 0（纯黑）到 100（纯白）。取决于目标物体在图像中的明暗，让目标物体的亮度做为你的L通道的范围里面
  - **`A`**: 表示颜色从绿（负值）到红（正值），范围 -128 到 127。
  - **`B`** : 表示颜色从蓝（负值）到黄（正值），范围 -128 到 127。
    - A通道和B通道可以囊括目标物体，同时也可以排除背景物体

  - 一般可以固定阈值，如果需要动态调整，可以依据色块函数返回的对象的一些参数去动态调整阈值
